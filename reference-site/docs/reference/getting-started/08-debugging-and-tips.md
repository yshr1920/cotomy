# Debugging and Tips

Practical notes for diagnosing UI issues.

## Goals

- Use standard browser tools effectively
- Keep the DOM-state model visible

## Tips

- [ ] Inspect actual DOM state instead of virtual state
- [ ] Check for duplicate listeners if behavior repeats
- [ ] Review scoped CSS in the document head
